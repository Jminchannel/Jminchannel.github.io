setup特点

setup语法糖

script标签添加 setup标记，不需要再写导出语句，默认会添加导出语句

<script setup>语法糖原理

reactive和ref函数

reactive

作用：接受对象类型数据的参数传入并返回一个响应式的对象

  1. 从vue包中导入reactive函数
  2. 在<script setup>中执行reactive函数并传入类型为对象的初始值，并使用变量接收返回值

ref

接收简单类型或者对象类型的数据传入并返回一个响应式的对象

二者对比

  1. 都是用来生成响应式数据

  2. 不同点

    1. reactive不能处理简单类型的数据
    2. ref参数类型支持更好，但是必须通过.value做访问修改
    3. ref函数内部的实现依赖于reactive函数

  3. 在实际工作中的推荐

    推荐使用ref函数，减少记忆负担

computed

计算属性基本思想和Vue2保持一致，组合式API下的计算属性只是修改了API写法

计算属性小案例

  • 计算属性中不应该有副作用，例如异步请求或者修改dom操作
  • 避免直接修改计算属性的值
  • 全选反选操作可以考虑

watch

单个数据或者多个数据监视

侦听一个或者多个数据的变化，数据变化时执行回调函数，两个额外参数 immediate控制立刻执行，deep开启深度侦听

immediate

希望在侦听器创建时立即触发回调，响应式数据变化之后继续执行回调

deep

deep深度监视，默认watch执行的是浅层监视

  • const ref1 = ref(简单类型) 可以直接监视
  • const ref2 = ref(复杂类型) 直接监视无法监视到复杂类型内部数据的变化

点击按钮，无反应！！所以需要开启deep

精确监听对象的某个属性

如何在不开启deep的前提下，侦听age的变化，只有age变化时才执行回调?
